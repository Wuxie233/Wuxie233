# Product

## Register

brand

## Users

AdventureX 参赛者、潜在队友与技术合作者。他们通常从招募帖进入 GitHub，需要在一分钟内判断 Wuxie 能负责什么、公开证据是否可信，以及合作责任如何划分。

## Product Purpose

把 GitHub Profile 变成一份可核验的个人作品集。页面必须先给出清晰定位，再用代表项目、系统边界和验收方式支撑定位，最后给出具体的组队入口。成功标准不是展示技术数量，而是让读者愿意打开项目并发起一次有效沟通。

## Brand Personality

清醒、精确、有主见。阅读感受应接近一份排版克制但观点鲜明的独立技术刊物，而不是简历模板、统计仪表盘或模型生成的个人介绍。

## Anti-references

- 霓虹紫蓝渐变、赛博光效和深色终端模板。
- GitHub Stats、语言占比、连续提交、访问量等动态卡片堆叠。
- 技术 Logo 墙、无上下文徽章和“全栈全能”式自我评价。
- 用上游功能、fork 提交量或 Agent 数量冒充个人成果。
- 只讲态度而不链接代码、系统边界和已知限制。

## Design Principles

1. 先让读者看懂责任，再让读者看技术。
2. 每个能力判断都连接到可访问的仓库或明确的系统结构。
3. 人负责判断、架构、集成和验收；Agent 的参与程度始终明示。
4. 用少量强视觉建立记忆点，不用组件数量制造丰富感。
5. 移动端仍能完成从定位、证据到联系的完整阅读路径。

## Accessibility & Inclusion

以 WCAG AA 对比度为最低要求。信息不能只依赖颜色；图片中的正文保持大字号和高对比度；所有视觉资产提供有效替代文本。页面不依赖动画、第三方服务或 JavaScript。
