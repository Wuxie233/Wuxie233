# Product

## Audience

产品开发者、工程师和潜在合作者。他们进入主页后，应能快速判断 Wuxie 做过哪些可用产品、具体负责什么，以及公开证据在哪里。

## Purpose

把 GitHub Profile 做成长期个人产品作品集。首页先展示真实产品，再补充工程项目与能力栈；不承担比赛招募、经历罗列或技术 Logo 展示。

## Content Contract

- 主展区固定为 Pulse、oc-remote、BDXSuperTool、FlyBuild。
- 工程选集固定为 micode、RimWorld AI、SpireVibePlaying。
- 产品说明只写已经核实的行为、责任和限制。
- fork 明确上游；私有源码明确标注，不用无法公开的代码作证据。
- GitHub 统计可汇总私有贡献，但不展示私有仓库名称或内容。
- B 站只展示账号与内容方向，不用粉丝数替代作品证明。

## Voice

短句、具体事实、第一人称责任。避免“真实问题”“系统边界”“交付闭环”等抽象套话反复出现，也不使用“全栈全能”“改变世界”一类自评。

## Success Criteria

1. 第一屏能看懂身份、工作范围和四个主产品。
2. 每个主产品都有可访问证据或明确标注的功能概览。
3. 在手机上仍能顺序读完产品、工程项目、能力栈和外部入口。
4. 页面不依赖第三方统计卡服务；每日统计失败时不会破坏现有页面。
