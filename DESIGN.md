# Design System

## Direction

明亮编辑部。以独立技术刊物的封面、目录页和系统图为参考，但使用几何无衬线排版，避免复古报纸、斜体衬线和常见“设计感”套件。整体像一份为一次重要组队会面准备的限量技术刊物。

## Color

- Paper: `#F5F4F0`
- Ink: `#111111`
- Signal red: `#F0442F`
- Soft gray: `#D9D7D0`
- White: `#FFFFFF`

红色只用于编号、状态、关键分隔和行动入口；正文始终使用黑色。所有图形同时依赖形状或文字表达意义。

## Typography

视觉资产使用系统可用的中文黑体与粗体无衬线。标题采用高字重、紧凑行距和正常字距；正文保持正常字重。英文技术标识可用等宽字体，但不用于大段正文。

## Composition

- 画布比例优先适配 GitHub README 的 1280px 内容宽度。
- 使用 12 列隐性网格和不对称留白，不使用圆角卡片阵列。
- 横幅只表达定位、责任和当前目标，不塞项目清单。
- 项目证据图一项目一张，结构图占主要面积，说明文字由 README 承担。
- 分区使用细规则线、实心色块和编号组织，不使用阴影、渐变或玻璃效果。

## README Structure

1. 定制横幅
2. 一段责任声明与当前状态
3. 代表项目证据图与短说明
4. 四步工作方式
5. AdventureX 组队条件与联系方式

## Asset Rules

- 全部资产随仓库存储，不依赖第三方统计服务。
- 图片包含清晰的英文或中英混排标题，最小字号需在 360px 宽度缩放后仍可辨认。
- 项目图只呈现仓库已证明的模块与数据流，不虚构产品界面、用户数量或线上规模。
