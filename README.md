## 我的工作方式

1. 先确认用户、问题、约束与验收证据，再决定技术方案。
2. 把可并行工作拆给不同 Agents，但保留清晰的接口和最终责任人。
3. 用构建、测试、日志、实际交互和失败路径验证结果。
4. 记录架构决策与已知限制，让下一轮开发不用重新猜测上下文。

Agent 是我的实现杠杆，不替代产品判断，也不替代我对结果负责。
